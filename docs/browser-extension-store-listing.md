## Extension name

- **Source:** Taken from the package **manifest** (not editable in the store form).
- **To change:** Update `name` in `browser-extension/src/manifest.ts`, then rebuild and re-upload the package.
- **Current value:** `Focus Bot`

---

## Description (required) — copy-paste this

Use this text in the store listing "Description" field:

```
Focus Bot helps you stay on task while browsing. Set a single task, and the extension shows you whether each page you visit is aligned with that task or distracting.

Features:
• Set a focus task and start a session from the popup or side panel.
• Real-time classification: each tab is labeled aligned or distracting using AI, with a brief reason.
• Session summaries: when you end a session, get a summary of time on aligned vs distracting pages.
• Daily analytics: view daily focus metrics and top sites in the side panel or analytics page.
• Distraction overlay: optional overlay on distracting pages to gently nudge you back.
• Exclude domains (e.g. work SSO) from classification.
• Works with your own OpenAI API key; your data is not sent to any third party except the API you configure.

Use Focus Bot for work (e.g. code review, research, docs) or for breaks (e.g. watching videos, social browsing)—you define the task, the extension helps you see how your browsing matches it.
```

## Chrome Web Store: Single purpose & permissions (copy-paste)

Use these in the developer dashboard under "Single purpose", "Permission justification", and "Remote code".

### Single purpose description

```
Focus Bot has one purpose: help you see whether your browsing matches a single task you set. You start a focus session with a task (e.g. "code review" or "research"). The extension classifies each page you visit as aligned or distracting with that task using AI, shows session summaries and daily analytics, and can display a gentle overlay on distracting pages. All logic runs from the extension package; the only external call is to the OpenAI API (using your own API key) for classification. No remote code is loaded or executed.
```

### storage justification

```
Used to persist user settings (API key, excluded domains), the active focus session (task, visits, timestamps), completed session history, last session summary, and last error message. All data stays in chrome.storage.local on the user's device. Required for the extension to work across popup, side panel, and browser restarts.
```

### tabs justification

```
Used to read the active tab's URL and title so the extension can classify the current page as aligned or distracting with the user's task; to detect tab changes and navigation; to send messages to the content script (e.g. show/hide distraction overlay); and to open the analytics page in a new tab. Required for real-time, per-tab focus tracking.
```

### sidePanel justification

```
Used to show the extension's analytics view and settings in the browser side panel. The side panel displays daily focus metrics, top sites, and configuration (API key, excluded domains). Users open it via the extension action or from the popup. Required for the documented side-panel experience.
```

### alarms justification

```
Used to run a periodic alarm (every minute) that updates the extension badge (e.g. session timer or status) and performs lightweight background checks. Keeps the UI and session state in sync without constant polling. Required for the badge and session timing behavior.
```

### idle justification

```
Used to detect when the user is idle (e.g. away from keyboard). When idle, the extension pauses the focus session and stops classifying pages so that inactive time is not counted and API calls are not made unnecessarily. When the user returns, the session resumes. Required for accurate session summaries and to avoid unnecessary OpenAI API usage.
```

### Host permission justification

```
Two host permissions are used:

1) https://api.openai.com/* — The extension sends the current page URL/title and the user's task text to the OpenAI API (using the user's own API key) to classify the page as aligned or distracting. No other external services are called.

2) <all_urls> — Required because: (a) the content script is injected on all pages to show the distraction overlay and receive classification results; (b) the extension must read the active tab's URL and title on any site the user visits to perform classification. No page content is sent to any server except the user-configured OpenAI API for classification.
```

### Are you using remote code?

**Select:** **No, I am not using remote code.**

**Justification** (paste into the form’s Justification field, 1,000 char max):

```
All JavaScript and assets are bundled in the extension package. The extension only makes HTTP requests to the OpenAI API (with the user's API key) to send and receive JSON for page classification. It does not load or execute any script or WebAssembly from the network. There are no external script tags, no eval() of external strings, and no dynamic import of remote modules. Remote code in Chrome’s sense is code that is fetched and executed at runtime from outside the package; this extension executes only code that is included in the uploaded package.
```

## Manifest vs store listing

- **Extension name** in the form is read from the **manifest** only; edit `browser-extension/src/manifest.ts` and re-pack to change it.
- **Description** in the form is the **store listing description** (long form); the manifest `description` is used elsewhere (e.g. install dialog). You can base the store text on the manifest description.

After changing the manifest, run in `browser-extension`:

```bash
npm run pack
```

Then upload the new `focusbot-extension.zip` and complete the listing fields (description, logo, tiles, screenshots, optional video).

## Privacy policy URL

The store requires a **Privacy policy URL** if the extension uses user data (e.g. web history, user activity). Focus Bot does, so you must provide a URL that serves the privacy policy.

**What’s in the repo:**

- **Source (Markdown):** `docs/focusbot-extension-privacy-policy.md` – edit this when you change the policy.
- **Hostable page (HTML):** `docs/focusbot-extension-privacy-policy.html` – same content as a single HTML file. Host this file at a public URL and use that URL in the store’s “Privacy policy URL” field.

**How to get a URL:**

1. **GitHub Pages** – Enable Pages for the repo (e.g. from the `main` branch, root or `/docs`). If the site is `https://<user>.github.io/focusbot/` and the file is at `docs/focusbot-extension-privacy-policy.html`, the URL is typically `https://<user>.github.io/focusbot/focusbot-extension-privacy-policy.html` (or `.../docs/focusbot-extension-privacy-policy.html` depending on your Pages source).
2. **Your own domain** – Upload the HTML file to your web server and use the full URL (e.g. `https://yoursite.com/focusbot-privacy.html`).
3. **Other static host** – Netlify, Cloudflare Pages, etc.: deploy the repo or upload the HTML and use the URL they give you.

After hosting, paste that URL into the Chrome Web Store (and Edge Add-ons) “Privacy policy URL” field.
